🎮 Sistema de Control

Algoritmos de control, navegación autónoma y arquitectura de software del proyecto CARTMAN

Introducción

El sistema de control de CARTMAN implementa múltiples niveles de control, desde el bajo nivel (control de motores) hasta el alto nivel (navegación autónoma y toma de decisiones). El software fue desarrollado utilizando una arquitectura modular que facilita el desarrollo, pruebas y mantenimiento.

Se implementaron algoritmos de control PID para estabilidad, odometría para localización, y algoritmos de planificación de trayectorias para navegación autónoma.

Control PID de Motores

📊 Controlador Proporcional-Integral-Derivativo

Se implementó un controlador PID para cada motor, permitiendo control preciso de velocidad y compensación de perturbaciones. Los parámetros fueron ajustados mediante el método de Ziegler-Nichols y refinados experimentalmente.

Ecuación del Controlador PID

u(t) = Kp × e(t) + Ki × ∫e(t)dt + Kd × de(t)/dt

Donde: e(t) = error, Kp = ganancia proporcional, Ki = ganancia integral, Kd = ganancia derivativa

// Pseudocódigo del controlador PID
error = setpoint - medicion;
integral += error * dt;
derivativo = (error - error_anterior) / dt;
salida = Kp*error + Ki*integral + Kd*derivativo;
error_anterior = error;

⚙️ Parámetros de Ajuste

  • Kp (Proporcional): X.XX - Respuesta rápida al error actual
  • Ki (Integral): X.XX - Elimina error en estado estacionario
  • Kd (Derivativo): X.XX - Amortigua oscilaciones y sobrepaso
  • Frecuencia de muestreo: XXX Hz
  • Límites de saturación: ±XXX PWM

Cinemática y Odometría

🚗 Modelo Cinemático

Se implementó el modelo cinemático diferencial para un robot de 4 ruedas, permitiendo calcular la velocidad lineal y angular en función de las velocidades de las ruedas izquierda y derecha.

v = (v_right + v_left) / 2
ω = (v_right - v_left) / L

📍 Odometría

Sistema de odometría basado en encoders para estimar la posición y orientación del robot. Se implementó integración numérica de las ecuaciones cinemáticas.

x += v * cos(θ) * dt
y += v * sin(θ) * dt
θ += ω * dt

Navegación Autónoma

El sistema de navegación autónoma integra múltiples algoritmos para planificación de trayectorias, detección de obstáculos y comportamientos reactivos.

🗺️

Planificación de Trayectorias

Algoritmo A* o RRT para encontrar rutas óptimas entre puntos en un mapa conocido.

👁️

Detección de Obstáculos

Fusión de sensores ultrasónicos y cámara para detección en tiempo real de obstáculos.

🔄

Comportamiento Reactivo

Respuestas inmediatas a eventos del entorno usando máquina de estados finitos.

🎯

Seguimiento de Trayectorias

Control de seguimiento para ejecutar las trayectorias planificadas con precisión.

Algoritmos Implementados

🔍 Algoritmo de Evitación de Obstáculos

  1. Leer sensores de distancia (ultrasónicos)
  2. Si distancia < umbral_seguridad:
  3.     Detener movimiento hacia adelante
  4.     Calcular dirección libre más cercana
  5.     Girar hacia dirección libre
  6. Si no hay obstáculos:
  7.     Continuar con trayectoria planificada
  8. Actualizar mapa local de obstáculos

🎯 Algoritmo de Seguimiento de Línea

  1. Capturar imagen de cámara
  2. Preprocesar imagen (escala de grises, umbralización)
  3. Detectar línea usando algoritmo de contornos
  4. Calcular error lateral respecto al centro
  5. Aplicar controlador PID para corrección de dirección
  6. Ajustar velocidades de motores izquierdo/derecho
  7. Repetir a frecuencia de XX Hz

🤖 Máquina de Estados para Navegación

  1. IDLE: Sistema en espera de comando
  2. NAVEGANDO: Siguiendo trayectoria planificada
  3. EVITANDO: Evadiendo obstáculo detectado
  4. REPLANIFICANDO: Recalculando ruta al destino
  5. DETENIDO: Obstáculo crítico o comando de parada
  6. OBJETIVO_ALCANZADO: Llegada exitosa al destino

Comunicación y Telemetría

📡 Protocolo de Comunicación

Se implementó un protocolo de comunicación bidireccional eficiente que permite enviar comandos desde la interfaz de usuario y recibir datos de telemetría del robot.

// Estructura de paquete de datos
struct TelemetryPacket {
  float velocidad_izq;
  float velocidad_der;
  float voltaje_bateria;
  float corriente;
  uint16_t distancia_sensores[4];
  float posicion_x, posicion_y, orientacion;
  uint8_t estado_sistema;
};

📈 Datos Transmitidos

  • Comandos (PC → Robot): Velocidades objetivo, modo de operación, configuración
  • Telemetría (Robot → PC): Estado de sensores, posición, orientación, diagnósticos
  • Video: Stream MJPEG o H.264 a 15-30 fps
  • Frecuencia de actualización: XX Hz para telemetría crítica

Pruebas y Validación

✅ Protocolo de Pruebas de Control

  • Pruebas unitarias: Validación de cada módulo de software individualmente
